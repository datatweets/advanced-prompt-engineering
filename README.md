This repository is a practical course on prompt engineering for business owners, managers, corporate employees, and students. It explains how to use generative AI for research, planning, writing, data analysis, review, and responsible decision support.

> [!IMPORTANT]
> AI output should be reviewed by a responsible human before it is used for decisions, public communication, customer data, legal work, finance, health, or safety-sensitive tasks.

## Course Goals

By the end of this course, learners should be able to:

1. Explain what generative AI can and cannot do, and how modern AI systems are structured.
2. Write clear prompts using the eight-element anatomy, nine sharpening techniques, and the bad-better-best pattern.
3. Use AI to support market research, audience analysis, and competitor review.
4. Create business content — posts, articles, emails, storyboards, and campaign plans — using structured multi-step workflows.
5. Build review steps that reduce hallucinations, privacy risks, bias, and poor-quality output.
6. Use AI for structured data analysis, generate insights from datasets, and produce actionable recommendations.
7. Design reusable prompt templates, team guidelines, and a practical team rollout plan.
8. Complete a capstone project that connects research, content, and governance.

## Suggested Schedule

The course contains approximately 12 hours of lessons and exercises across four modules (including bonus lessons). It works well as a two-day workshop, a self-paced online course, or a series of team learning sessions.

| Format | Suggested structure |
| --- | --- |
| Two-day workshop | Day 1: Modules 1 and 2. Day 2: Modules 3 and 4 with Capstone planning. |
| Self-paced | One module per week over four weeks. Complete exercises before moving on. |
| Team learning | One lesson per session, three sessions per module, one module per week. |

### Day 1 — Foundations and Research

| Session | Outcome |
| --- | --- |
| Module 1 / Lessons 1–2: AI foundations, prompt anatomy, and nine techniques | Understand what AI does and write structured, sharpened prompts |
| Module 1 / Lesson 3: Bad-better-best pattern and ask-me-first technique | Build, test, and improve prompts step by step |
| Module 2 / Lessons 1–2: Audience research and competitor analysis | Turn business questions into grounded research; compare competitors with source material |
| Module 2 / Lesson 3: Customer personas | Build evidence-based persona cards and map them to content needs |

### Day 2 — Content and Strategy

| Session | Outcome |
| --- | --- |
| Module 3 / Lessons 1–2: Blog posts and social media content | Draft long-form and social content using structured multi-step workflows |
| Module 3 / Lesson 3: Digital storyboards | Plan visual and narrative flow for a short campaign video |
| Module 3 / Bonus: Data analysis with ChatGPT | Use AI for structured data analysis and insights generation |
| Module 4 / Lessons 1–2: Responsible AI use and content distribution | Apply risk review, privacy rules, and goal-based channel planning |
| Module 4 / Lesson 3: Team guidelines and rollout | Draft one-page guidelines and a five-step team rollout plan |
| Capstone planning | Combine all skills into a complete project brief |

## How To Use This Repository

Start with Module 1 if you are new to AI. If you already use AI tools, skim Module 1 and spend more time on the exercises in Modules 2, 3, and 4.

Bonus lessons (image generation and data analysis) are optional but provide valuable additional skills for specific use cases.

Each lesson uses this pattern:

| Label | Meaning |
| --- | --- |
| Concept | The idea learners need to understand |
| Scenario | A realistic business or classroom situation |
| Prompt Pattern | A reusable prompt structure |
| Practice | A short activity learners can complete |
| Review | A quality check before using the output |
